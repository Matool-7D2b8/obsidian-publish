# 笔记格式渲染错误

**日期**：2026-05-25  
**状态**：已解决

## 现象

1. 数学笔记 `02_一元微分学/高阶导数求导` 网页格式混乱
2. 幂函数分段公式显示 KaTeX 报错
3. 部分公式根本无法渲染（14 个 → 70 个）

## 根因分析

### 原因一：cases 环境的 `&` 对齐符被 HTML 转义

**影响**：`\begin{cases}...\end{cases}` 中的 `&` 被 Quartz markdown 预处理器转义为 `&amp;`，KaTeX 无法解析。

```
原：\begin{cases} ... , & n \le m \\ 0, & n > m \end{cases}
被转义为：\begin{cases} ... , &amp; n \le m \\ 0, &amp; n > m \end{cases}
→ KaTeX ParseError
```

**修复**：移除 `cases` 环境，改为两个独立公式。
```
改前：$$(x^m)^{(n)} = \begin{cases} ... , & n \le m \\ 0, & n > m \end{cases}$$
改后：$$(x^m)^{(n)} = ... \quad (n \le m)$$  $$(x^m)^{(n)} = 0 \quad (n > m)$$
```

### 原因二：`---` 被 markdown 转为破折号

**影响**：`$$...---...$$` 中的 `---` 被转为 `——`（em-dash），破坏数学模式。

**修复**：`---` → `\implies`

### 原因三：`\text{}` 中的 Unicode `∑` 不兼容

**影响**：`\text{∑ 形式：}` 导致 8 个公式全部报错。

**修复**：`\text{∑ 形式：}` → `\text{求和形式：}`

### 附带问题：git 失效代理阻断推送

**影响**：obsidian-git 无法推送笔记到 GitHub。

**详细**：git 全局配置了代理 `https.proxy=http://127.0.0.1:7890`，但该代理未运行，导致所有 git push 失败（`Connection was reset`）。

**修复**：
```bash
git config --global --unset https.proxy
```

通过直连 GitHub 解决。

### 附带问题：GitHub Actions `GH_PAT` secret 失效

**修复**：通过 GitHub API 更新了 Secret。

## 总结

| 修复项 | 文件 | 改动 |
|--------|------|------|
| cases 环境 | `Math_note/Math/02_一元微分学/高阶导数求导.md` | 拆为两个独立公式 |
| `---` 破折号 | 同上 | 改为 `\implies` |
| Unicode ∑ | 同上 ×8 | 改为中文 `求和形式：` |
| git 代理 | `~/.gitconfig` | 移除 `https.proxy` |
| GH_PAT | GitHub Secrets | 更新为新 token |

## 写作建议（避免格式问题）

- `cases`/`array`/`aligned` 中不要用 `&`，用 `\quad` 代替
- 数学公式内不要用 `---`，用 `\implies` 或 `\to`
- `\text{}` 中避免 Unicode 数学符号，直接用中文
