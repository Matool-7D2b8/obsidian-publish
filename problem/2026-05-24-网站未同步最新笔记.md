# 网站未同步最新笔记

**日期**：2026-05-24  
**状态**：已解决

## 现象

1. 在 Obsidian 中新增了 `Data_Struct/08_排序` 章节，但网站没有显示
2. 首页链接跳转 404
3. 笔记中的图片无法加载

## 根因分析

### 原因一：obsidian-git 不自动 push

**影响**：笔记只 commit 到本地，未推送到 GitHub，GitHub Actions 构建不到新内容。

**详细**：三个 vault 的 `autoPushInterval` 均设为 `0`（禁用自动推送）。obsidian-git 每 5 分钟自动 commit，但 commit 只保存在本地 `.git` 中，从不推送到远程仓库。

**修复**：
```
文件：各 vault 的 .obsidian/plugins/obsidian-git/data.json
改动：autoPushInterval: 0 → autoPushInterval: 5
```

现在笔记每 5 分钟自动 commit 并 push 到 GitHub。

### 原因二：nginx 缓存 contentIndex.json

**影响**：即使部署成功，浏览器仍展示旧的侧边栏目录树，新章节不可见。

**详细**：nginx 对 `/static/` 目录设置了 30 天缓存（`expires 30d`）。`contentIndex.json` 位于 `/static/` 下，浏览器长期使用缓存的旧版文件，其中的条目不包含新章节。

**修复**：
```
文件：/etc/nginx/sites-available/obsidian
改动：为 /static/contentIndex.json 添加独立 location 块，设置 Cache-Control: no-cache, must-revalidate
```

### 原因三：首页链接路径错误（附带修复）

**现象**：首页"浏览数据结构笔记 →"跳转 404。

**详细**：同步脚本 `sync-content.sh` 去掉了 vault 顶层包装目录（`DS/`、`Math/`），但 `content/index.md` 中的链接仍包含这些目录。

**修复**：
- `/Data_Struct/DS/01_绪论/...` → `/Data_Struct/01_绪论/...`
- `/Math_note/Math/01_极限/...` → `/Math_note/01_极限/...`

### 原因四：图片路径解析错误（附带修复）

**现象**：笔记中的图片无法加载，路径被错误转换为 `../../assets/...`。

**详细**：`quartz.config.ts` 中 `CrawlLinks` 插件使用 `markdownLinkResolution: "shortest"`，对中文目录名处理有 bug，导致图片相对路径错误。

**修复**：`markdownLinkResolution` 改为 `"relative"`。

## 总结

| 修复项 | 文件 | 改动 |
|--------|------|------|
| 自动 push | `.obsidian/plugins/obsidian-git/data.json` ×3 | `autoPushInterval: 0→5` |
| 缓存策略 | `/etc/nginx/sites-available/obsidian` | contentIndex.json no-cache |
| 首页链接 | `content/index.md` | 去掉 DS/、Math/ 包装路径 |
| 图片路径 | `quartz.config.ts` | `shortest` → `relative` |

## 同步流程（修复后）

```
Obsidian 写笔记 → obsidian-git 5min 内 commit + push → GitHub Actions 15min 内构建部署 → 网站更新
```

用户无需任何手动操作。
